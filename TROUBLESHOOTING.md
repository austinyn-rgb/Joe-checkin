## GitHub 问题

### ❓ 无法创建仓库

**症状**：点击创建后报错或没有反应

**解决方案**：
1. 检查仓库名称是否重复（换个名字试试）
2. 确认已登录 GitHub
3. 刷新页面后重试
4. 清除浏览器缓存

---

### ❓ 上传文件失败

**症状**：上传文件时一直转圈或报错

**解决方案**：
1. 检查文件大小（GitHub 限制单个文件 100MB）
2. 确认 ZIP 文件完整（重新压缩）
3. 尝试使用 GitHub Desktop 上传
4. 检查网络连接

---

### ❓ Push 代码失败

**症状**：`git push` 时报错

**解决方案**：
1. 检查是否已登录：`git config --global user.name` 和 `git config --global user.email`
2. 重新输入 GitHub 账号密码
3. 如果使用 token，确认 token 有效
4. 尝试强制推送：`git push -f origin main`（慎用）

---

## Vercel 问题

### ❓ 部署失败 - Build Error

**症状**：部署时报错 `Build Error`

**解决方案**：
1. 检查 `package.json` 是否存在
2. 确认 `src/app/page.tsx` 文件存在
3. 查看 Vercel 部署日志，找到具体错误
4. 检查代码语法错误

---

### ❓ 部署失败 - Install Error

**症状**：部署时报错 `Install Error`

**解决方案**：
1. 确认使用了 `pnpm` 作为包管理器
2. 检查 `package.json` 中的依赖是否正确
3. 尝试删除 `node_modules` 和 `pnpm-lock.yaml`，重新安装
4. 在本地运行 `pnpm install` 测试是否成功

---

### ❓ 网页显示 404

**症状**：打开网页后显示 `404: Page Not Found`

**解决方案**：
1. 确认 Vercel 部署成功（绿色对勾）
2. 检查访问的 URL 是否正确
3. 清除浏览器缓存
4. 在 Vercel 控制台查看部署状态
5. 重新部署：在 Vercel 中点击 "Redeploy"

---

### ❓ 网页显示空白

**症状**：打开网页后一片空白

**解决方案**：
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页是否有错误
3. 检查 `src/app/page.tsx` 文件内容是否完整
4. 确认没有语法错误
5. 重新部署应用

---

## 网页使用问题

### ❓ 无法点击复选框

**症状**：点击复选框没有反应

**解决方案**：
1. 确认输入框已填写内容
2. 如果内容为空，会提示"请先填写内容"
3. 先填写内容，再点击复选框

---

### ❓ 提交打卡没有反应

**症状**：点击"提交打卡"按钮后没有反应

**解决方案**：
1. 确认至少勾选了一项活动
2. 确认所有勾选的活动都填写了内容
3. 检查浏览器控制台是否有错误（F12）
4. 刷新页面后重试

---

### ❓ 积分计算错误

**症状**：积分显示不正确

**解决方案**：
1. 每项活动获得 100 积分
2. 完成几项就获得几项的积分（如 3 项 = 300 积分）
3. 检查历史记录中的完成项数和积分是否匹配
4. 刷新页面重新计算

---

### ❓ 历史记录不显示

**症状**：没有显示历史记录

**解决方案**：
1. 历史记录卡片始终显示
2. 如果没有打卡记录，会显示"还没有打卡记录"
3. 完成一次打卡后，历史记录会自动显示
4. 检查是否清除了浏览器数据（数据会丢失）

---

### ❓ 导出功能不工作

**症状**：点击导出按钮没有反应或下载失败

**解决方案**：
1. 确认有打卡数据（至少一条记录）
2. 检查浏览器是否允许下载
3. 检查浏览器控制台是否有错误（F12）
4. 尝试使用其他浏览器（Chrome, Safari, Edge）
5. 检查网络连接

---

## 数据问题

### ❓ 数据丢失了

**症状**：打卡记录突然消失

**原因**：
- 清除了浏览器数据
- 使用了隐私/无痕模式
- 更换了设备或浏览器

**解决方案**：
- 数据无法恢复（当前版本不支持导入）
- 定期导出数据备份（JSON 格式）
- 重新开始打卡

**预防措施**：
1. 每周导出一次数据备份
2. 保存到云盘或电脑
3. 不要清除浏览器数据

---

### ❓ 数据不会跨设备同步

**症状**：在 iPad 上打的卡，在电脑上看不到

**原因**：数据保存在设备浏览器中，不跨设备同步

**解决方案**：
- 每个设备需要独立打卡
- 如需同步，需要使用后端数据库（需要额外开发）
- 定期导出数据，手动合并

**替代方案**：
- 固定在一台设备上使用（如 iPad）
- 每周导出数据，分享给家长查看

---

### ❓ 无法导入数据

**症状**：导出了数据，但无法重新导入

**原因**：当前版本不支持数据导入功能

**解决方案**：
- 只能重新开始打卡
- 如需导入功能，可以联系开发者添加

---

## 飞书分享问题

### ❓ 链接无法在飞书中打开

**症状**：点击链接后无法打开网页

**解决方案**：
1. 检查链接是否完整复制
2. 尝试复制到浏览器中打开
3. 检查网络连接
4. 确认网页部署成功（在浏览器中测试）

---

### ❓ 飞书无法复制链接

**症状**：点击"复制链接"按钮后提示失败

**解决方案**：
1. 手动复制浏览器地址栏的链接
2. 粘贴到飞书中
3. 检查浏览器权限设置
4. 尝试使用其他浏览器

---

### ❓ 飞书中无法正常使用

**症状**：在飞书中打开网页后功能异常

**原因**：飞书内置浏览器可能有兼容性问题

**解决方案**：
1. 点击右上角"在浏览器中打开"
2. 在 Safari 或 Chrome 中使用
3. 添加到主屏幕，像 App 一样打开

## 其他问题

### ❓ 页面加载速度慢

**症状**：打开网页需要很长时间

**解决方案**：
1. 检查网络连接
2. Vercel 使用全球 CDN，第一次加载可能慢
3. 加载后会缓存，后续速度会更快
4. 清除浏览器缓存后重试

---

### ❓ 无法修改应用名称

**症状**：想要修改"杨子祐寒假打卡攒积分"这个标题

**解决方案**：
1. 修改 `src/app/page.tsx` 中的标题文字
2. 提交到 GitHub
3. Vercel 自动更新（1-3 分钟）

**注意**：需要修改代码，需要一点技术基础。

---

### ❓ 无法修改积分规则

**症状**：想要修改积分规则（如每项 200 分）

**解决方案**：
1. 修改 `src/app/page.tsx` 中的积分计算逻辑
2. 将 `100` 改为 `200`
3. 提交到 GitHub
4. Vercel 自动更新

**注意**：修改只影响未来的打卡，历史记录的积分不会改变。

---

### ❓ 免费额度会用完吗？

**症状**：担心 Vercel 免费额度不够用

**答案**：
- Vercel 免费版包含：每月 100GB 带宽、6,000 分钟构建时间
- 家庭使用完全够用
- 假设每天打卡 1 次，每月访问 30 次，带宽使用 < 1GB
- **结论**：不会用完

---

### ❓ 需要服务器吗？

**症状**：担心需要购买服务器

**答案**：
- 不需要购买服务器
- Vercel 提供免费的托管服务
- 自动扩容，自动部署

---

### ❓ 可以多人使用同一个应用吗？

**症状**：想知道是否可以为多个孩子创建

**答案**：
- 可以，但每个人的数据是独立的
- 每个设备的数据互不干扰
- 如果需要多个孩子分别打卡，可以：
  - 方式 1：每人用不同的设备
  - 方式 2：修改应用名称，创建多个独立的应用

## 💡 预防措施

为了避免遇到问题，建议：

1. **定期备份数据**
   - 每周导出 JSON 格式数据
   - 保存到云盘或电脑

2. **不要清除浏览器数据**
   - 打卡记录保存在浏览器中
   - 清除数据会丢失所有记录

3. **固定使用设备**
   - 建议固定在一台设备上使用（如 iPad）
   - 避免跨设备使用导致数据不同步

4. **定期测试功能**
   - 每周测试一次打卡功能
   - 确认数据正常显示
   - 测试导出功能是否正常

5. **更新代码前备份**
   - 如果要修改代码，先导出数据
   - 避免代码更新导致数据丢失
